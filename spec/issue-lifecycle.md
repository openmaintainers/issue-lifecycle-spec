# Formal Lifecycle Specification

## State Definitions

### New
Issue has been created but not yet reviewed.

### Triaged
Issue has been categorized and assessed.

### Accepted
Issue is approved for work.

### In Progress
Work has started.

### Blocked
Progress is blocked by dependencies or decisions.

### Needs Review
Implementation complete; awaiting review.

### Resolved
Solution accepted and merged.

### Closed
Finalized and no further action required.

### Archived
Historical or stale issue preserved for reference.

## State Transitions

New -> Triaged
Triaged -> Accepted
Accepted -> In Progress
In Progress -> Needs Review
Needs Review -> Resolved
Resolved -> Closed

Optional:
Any -> Blocked
Closed -> Archived
