# Issue Lifecycle Spec

A formal specification for the lifecycle states of issues in open source repositories.

This specification provides a consistent framework for issue management across projects, improving triage, transparency, analytics and contributor experience.

## Goals

- Standardize issue progression
- Improve maintainability
- Enable automation tooling
- Support ecosystem-wide analytics

## Lifecycle States

1. New
2. Triaged
3. Accepted
4. In Progress
5. Blocked
6. Needs Review
7. Resolved
8. Closed
9. Archived
